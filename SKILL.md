# Execution Framework v3.2

**Role change:** The agentic harness has been replaced by the `work-principles`
plugin (phase gates + tool-triggered auto-loading).  The execution-framework
now exists purely as a manual keyword-based recommendation tool.

## How to Use (Manual)

```bash
# Get a skill recommendation for a task
python3 scripts/recommend.py "帮我把 Nginx 配置成反向代理"

# JSON output for scripting
python3 scripts/recommend.py --json "push this repo"

# Interactive mode
python3 scripts/recommend.py --interactive
```

## Architecture (3 Layers)

```
recommend.py "<task>"     (🔍 recommend)
    │
    ├── workflow:  Ordered steps
    └── background: Behavioral rules
    │
    ▼
skills_index.json          (📇 index, generated by --scan)
    │
    ▼
sync-routing.py --scan     (📡 discovery, scans config.yaml paths)
```

## Background Capabilities

| 纪律 | 模式 | 生效相位 | 规则 |
|:-----|:-----|:---------|:-----|
| `work-principles` | behavioral | all phases | 诚实优先、步骤透明、先研究再动手、先保全再改 |
| `change-safeguard` | active | modifying | 每次修改前备份、修改后五维扫描 |
| `vcs-discipline` | active | executing | Conventional Commits、GPG 签名、无 WIP commit |

## Self-Evolution Pipeline

1. **Discovery** — `sync-routing.py --scan` reads paths declared in `config.yaml`, scans skill directories, generates `skills_index.json`
2. **Index** — `skills_index.json` stores skill metadata (name, tags, indicators, description)
3. **Recommend** — `recommend.py` reads `routing.yaml` + `skills_index.json`, matches task descriptions against indicators, returns sorted skill chain

To rebuild the index:
```bash
cd ~/.astra/repos/astra-skill-execution-framework
python3 scripts/sync-routing.py --scan --write-index
```

## Pitfalls

1. **routing.yaml is the single source of truth** — SKILL.md tables are snapshots. Routing decisions always follow YAML.
2. **No auto-trigger** — The agent is never forced to call this tool. The harness uses `work-principles` plugin gates instead.
3. **Keyword-only** — Recommendations are based on keyword matching, not semantic understanding. Use as a hint, not as law.
4. **Manual debugging tool** — This is a fallback for when the agent needs a quick suggestion, not the primary workflow.
